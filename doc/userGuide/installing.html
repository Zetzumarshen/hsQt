2.1. Introduction and System Requirements

qtHaskell consists of two components, a C++ layer built using the Qt build tool qmake and a Haskell component built using the Haskell build tool Cabal. qtHaskell requires Qt (4.3.1 or later) and GHC (version 6.8.2 or higher) to be available on any system in which it is installed. Version 1.1.4 of qtHaskell has been tested with versions 4.3.1 and later of Qt and with versions 6.8.2 and later of GHC on Linux and Windows systems. The previous version of qtHaskell was reported as working without any problems on MacOS, but users are advised that at release, version 1.1.4 has been tested only as described above. Feedback from users of other environments is welcome.

N.B. The qtHaskell package name has changed from "qtc" to "qt" in this release. The "qtc" package for qtHaskell-1.1.2 must therefore be unregistered using ghc-pkg before starting the build procedure for qtHaskell-1.1.4.

2.1.1. Perl

The recommended method for building qtHaskell-1.1.4 is to use the build script included with this release. This is a cross-platform Perl script which requires version 5.10 (or later) of the Perl language. This a later version of Perl than is provided with current GHC distributions.

A suitable version of Perl for all major systems is available for free. See www.perl.org/get.html. Strawberry Perl is the recommended version for Windows builds of qtHaskell.

2.1.2. Haskell Dependencies

QtHaskell-1.1.4 requires the HOpenGL Haskell bindings for OpenGL.

N.B. If building with dynamic linking enabled with the --enabled-shared paramater on Linux systems (this is not the default mode), it is essential that the dependent libraries have also been configured with --enable-shared and built and installed for the version of GHC in use with qtHaskell.

2.1.2.1. OpenGL Dependencies

On Linux/X11 systems Qt normally links to a 3rd party OpenGL implementation (e.g. Mesa3D) and on Windows it normally uses its own OpenGL implementation. In any case, Qt does not officially support version 3.x of the OpenGL specification. On the other hand versions 2.3 and higher of HOpenGL, the standard Haskell OpenGL bindings are based on the 3.x OpenGL specification. If qtHaskell is built with version 2.3 or higher of HOpenGL this will result in very poor performance of some OpenGL based applications on Linux systems and probable run-time errors on Windows and other systems. It is recommended therefore that while building the qtHaskell library and subsequent qtHaskell OpenGL applications, version 2.2.x or earlier of HOpenGL be installed or visible as the current Haskell OpenGL package. In this case qtHaskell OpenGL applications should be trouble free and highly efficient. Since Qt is reported as having some degree of compatibility with version 3 of OpenGL, an upper limit on HOpenGL versions is not enforced in the qtHaskell build process. It is nonetheless highly recommended that only version 2.2.x or less of HOpenGL be used with qtHaskell version 1.1.4.

2.1.2.2. Parallel Dependencies

The extra package qt-glome requires the Parallel library.

2.1.3. Building with Qt verion 4.6.x or higher on Windows systems

Qt version 4.6.x or higher on Windows requires version 4.4 of the Gnu C++ compiler and associated build tools. These are provided with Qt version 4.6.x or higher for Windows. They are not compatible with the MinGW tools provided with the GHC distributions for Windows.

With version 6.12.x or higher of GHC this problem can be overcome as follows. Simply copy the MinGW directory of your Qt distribution over the MinGW directory of your GHC distribution, and proceed with your qtHaskell installation as normal.

This is not possible with earlier versions of GHC (i.e. 6.10.x or earlier). It is therefore essential in order to build qtHaskell-1.1.4 with Qt version 4.6.x or higher on Windows, that you use GHC 6.12.x or higher and follow the above instructions.

N.B. you must make sure your path environment variable is set to the correct versions of MinGW, Haskell and Qt. You can test this prior to starting the build script by typing

g++ --version

qmake --version

ghc --version

You should check in particular that the version of g++ is 4.4 otherwise the Qtc wrapper libraries will not link correctly.
