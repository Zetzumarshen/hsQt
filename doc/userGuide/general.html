2.2. General Instructions

The qtHaskell version 1.1.4 distribution file qtHaskell-1.1.4.x.tar.gz should be expanded in some suitable directory hereafter referred to as myqthlocation. "cd" to the directory myqthlocation/qtHaskell-1.1.4. QtHaskell can then be built on most Linux systems using the bash script

./build

or on Windows systems by running ("as Administrator" on Vista) the batch file

build

Both these commands call the qtHaskell cross-platform Perl build script in its default mode with no options. The Perl script can also be run in this mode with the command

perl build.pl

or on Linux systems

./build.pl

This script will build both the C++ component and the Haskell component of qtHaskell and also compile all the example applications contained in the directories myqthlocation/qtHaskell-1.1.4/examples and myqthlocation/qtHaskell-1.1.4/demos.

The build script is highly controlable, allowing every major phase and subphase of the build to be enabled or disabled. It should however build qtHaskell without any options on Linux and Windows systems (MacOS may require one option for correct linking). A full list of the build script options syntax and semantics is in the section:


      Chapter 4, Using the Perl Build Script
      

In particular, on Linux systems, if it is required to build qtHaskell without calls to the sudo command from the build script, the required options to achieve this are detailed in the section:


      Section 4.5.3, “Build qtHaskell Without sudo”
      

Also, on Linux systems, users of GHC version 6.12.x or higher can build qtHaskell with dynamic linking enabled. This is not the default mode, the --enable-shared option must be specified and all dependency pacakges must be built (or rebuilt) with dynamic linking enalbed (i.e. with the --enable-shared parameter specified on the cabal build command). See the section:


      Section 4.4.41, “shared”
      

In cases where the build script is not applicable or fails to work for some reason, please refer to the appropriate manual instruction sequences given in the following two sections:


      Section 2.3.1, “Linux Manual Installation”
      Section 2.3.2, “Windows Manual Installation”
      

The Linux instructions should be very close to what is required on most Unix type systems. This includes MacOS, but MacOS users should note the following points:

1. The Qt qmake tool used to generate the Makefiles used to build the Qtc wrapper libraries does not generate Makefiles by default on MacOS. The option -spec macx-g++ is required. This is used by the build script and must be used in any manual builds.

2. The build script parameter --extra-ld-option=--architecture=x86_64 may be required. See the section


      Section 4.4.43, “extra-ld-opt”
      

for short forms of this option.

N.B. on Windows systems, the default build script does not link the library file HSqt-1.1.4.o, since this takes a very long time (3 hours on a 2.8GHz Pentium 4). This file is needed by GHCi, which is probably required by most experienced Haskellers. To enable building of this file include the option --enable-ld when calling the build script, but be advised that the Haskell build phase may take several hours in this case.
